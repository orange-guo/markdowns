可靠, 可扩展与可维护的应用系统

数据密集型应用的主要模块​

  • 数据库
  • 缓存
  • 索引
  • 流式处理
  • 批处理

数据系统​

三个主要问题​

  • 可靠性 出现问题时可以继续工作
  • 可扩展性 随着规模增加可以以一种方式来匹配这种增长
  • 可维护性 可以很
  • 方便的修改和扩展

可靠性​

对于可靠性的主要有以下期望

  • 程序执行用户所期望的功能
  • 可以容忍用户出现错误或不正确的使用方法
  • 性能可以应对典型场景， 合理负载压力和数据量
  • 系统可以防止任何未经授权的访问额滥用
